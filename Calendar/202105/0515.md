### 싱글톤 패턴 (Singleton Pattern)

- 정의
    - 어떤 클래스가 최초 한 번만 메모리를 할당하고, 그 메모리에 객체를 만들어 사용하는 것
    - 객체를 하나만 생성하도록 하며, 생성된 객체를 어디에서든지 참조할 수 있도록 함
- 장점 (사용하는 이유)
    - 한 번의 객체 생성으로 고정된 메모리 영역을 얻어 재사용을 할 수 있기 때문에 메모리 낭비를 방지함
    - 전역성을 띄어서 다른 클래스의 인스턴스와 데이터 공유가 용이함
    - 인스턴스가 절대적으로 한 개만 존재하는 것을 보증하므로 개발 시 실수를 줄일 수 있음
    - 싱글톤을 상속시킬 수 있음
    - DBCP처럼 공통된 객체를 여러 개 생성해서 사용해야 하는 상황에서 사용 가능
    - 두 번째부터는 객체 로딩 시간이 현저하게 줄어들어 성능이 좋음
- 문제점
    - 싱글톤 인스턴스가 너무 많은 일을 하거나 많은 데이터를 공유할 경우, 싱글톤 객체를 사용하는 다른 객체들 간의 결합도가 높아져 객체지향 설계 원칙의 '개방-폐쇄 원칙'을 위배하게 됨
        - 따라서 수정 및 테스트가 어려워짐
    - 멀티 스레드 환경에서 동기화 처리를 안 할 경우 인스턴스가 2개 생성될 수 있음
        - 경합 조건(Race Condition)을 발생시키는 경우에 해당
        - 경합 조건 : 메모리와 같은 동일한 자원을 2개 이상의 스레드가 이용하려고 경합하는 것
    - 싱글톤 객체를 수정할 경우 싱글톤 객체를 사용하는 곳에서 사이트 이펙트가 발생할 수 있음
    - 객체의 파괴 시점을 컨트롤하기 어려움
    - 전역변수보다 사용하기가 불편함
- 구현 방법
    - 여러 가지가 존재하는데, 무엇보다도 Thread-safe 하게 구현하는 것이 중요
    - 가장 많이 쓰는 방식은 Holder에 의한 초기화 방식
